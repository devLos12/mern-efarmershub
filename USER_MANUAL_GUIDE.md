# ADMIN MANUAL

## Dashboard Overview
**Purpose**: Central hub for platform monitoring and management

### Step-by-Step: First Time Login & Dashboard Navigation
1. Open the admin login page
2. Enter your admin email and password
3. Click "Login" button
4. You will be directed to the Admin Dashboard
5. You can see:
   - **Total Users**: Number of registered users on platform
   - **Total Sales**: Total revenue generated
   - **Total Orders**: Number of orders placed
   - **Sales Graph**: Visual representation of sales trends
6. From the dashboard, use the **Sidebar Menu** to navigate to different modules

## Module 1: Account Management

### Sub-Module 1.1: View & Manage All Accounts

**Step-by-Step: Viewing All User Accounts**
1. From Admin Dashboard, click **"Accounts"** in the sidebar menu
2. You will see a list of all user accounts (sellers, riders, users, other admins)
3. For each account, you can see:
   - User name
   - Email address
   - Account type (Seller/Rider/User/Admin)
   - Verification status (Verified/Pending)
   - Account action buttons
4. Use the **search bar** to find specific users by name or email
5. Use **filters** to sort by account type or verification status

**Step-by-Step: View User Profile Details**
1. From the Accounts list, click on a user's name or **"View Profile"** button
2. You will see detailed information:
   - Full name
   - Email address
   - Contact number
   - Account type
   - Verification status
   - Profile image
   - For Riders: License image and Plate number image
3. Review the account information
4. Click **"Back"** to return to accounts list

### Sub-Module 1.2: Add New Account (Seller/Rider/Admin)

**Step-by-Step: Adding a New Admin Account**
1. From Accounts page, click **"Add New Account"** button
2. A form will appear asking for:
   - Full name
   - Email address
   - Password
   - Account type (select Admin)
   - Contact number (optional)
3. Fill in all required fields with correct information
4. Click **"Submit"** button
5. System will verify the email and create the account
6. A success message will appear: "Account created successfully"
7. The new admin can now login with their email and password

**Step-by-Step: Adding a New Seller Account**
1. From Accounts page, click **"Add New Account"** button
2. In the form, select account type as **"Seller"**
3. Fill in the form fields:
   - Full name
   - Email address
   - Password
   - Contact number
   - Business name (optional)
4. Click **"Submit"** button
5. An email verification will be sent to the seller
6. Seller must verify their email before they can start uploading products
7. You will see a success notification

**Step-by-Step: Adding a New Rider Account**
1. From Accounts page, click **"Add New Account"** button
2. In the form, select account type as **"Rider"**
3. Fill in the required fields:
   - Full name
   - Email address
   - Password
   - Contact number
   - License Image (upload file)
   - Plate Number Image (upload file)
4. Upload the rider's identification and vehicle plate images
5. Click **"Submit"** button
6. System will save the account and verify the documents
7. You will see a success message

### Sub-Module 1.3: Update Account Verification Status

**Step-by-Step: Approve/Verify User Account**
1. From Accounts list, find the user with **"Pending"** status
2. Click the user's account to view details
3. Click **"Update Verification Status"** button
4. Select **"Approved"** or **"Verified"** from the dropdown
5. Click **"Confirm"** button
6. A success message will show the account is now verified
7. The user will receive a notification that their account is verified

**Step-by-Step: Update Rider Profile Images**
1. From Accounts list, click on a Rider's account
2. Click **"Edit Profile"** button
3. You can update:
   - License image (upload new file)
   - Plate number image (upload new file)
   - Contact information
4. Click **"Save Changes"** button
5. System confirms the update
6. Changes are now applied to rider's profile

### Sub-Module 1.4: Remove Account

**Step-by-Step: Delete User Account**
1. From Accounts list, find the account you want to remove
2. Click the **"Delete/Remove"** button (usually represented by trash icon)
3. A confirmation dialog will appear: "Are you sure you want to remove this account?"
4. Click **"Yes, Remove"** to confirm
5. The account will be deleted from the system
6. A success message shows: "Account removed successfully"
7. The deleted user will no longer be able to login

## Module 2: Product Management

### Sub-Module 2.1: View All Products

**Step-by-Step: Browsing Platform Products**
1. From Admin Dashboard, click **"Products"** in the sidebar
2. You will see a list of all products from all sellers
3. For each product, you can see:
   - Product name
   - Price
   - Stock quantity
   - Seller name
   - Product category
   - Product image
   - Approval status
4. Use the **search bar** to find products by name
5. Use **filters** to sort by:
   - Category (Fruits, Vegetables, Root Crops, etc.)
   - Price range
   - Seller name
   - Stock status (In Stock / Out of Stock)
   - Status (Approved / Pending / Rejected)

### Sub-Module 2.2: View Product Details

**Step-by-Step: Viewing Detailed Product Information**
1. From Products list, click on a product or **"View Details"** button
2. You will see complete product information:
   - Product name
   - Description
   - Price per unit
   - Stock quantity
   - Weight (in kg)
   - Category
   - Lifespan (days until expiry)
   - Seller information (name, contact, email)
   - Product image
   - Product approval status
   - Upload date
3. You can see the **edit** and **delete** options at the bottom
4. Click **"Back"** to return to products list

### Sub-Module 2.3: Update/Edit Product Details

**Step-by-Step: Editing Product Information**
1. From Products list, click **"View Details"** for the product you want to edit
2. Click **"Edit Product"** button
3. A form will appear with current product information
4. You can modify:
   - Product name
   - Price
   - Stock quantity
   - Category
   - Description
   - Lifespan (days)
   - Product image (upload new image if needed)
5. Make your changes
6. Click **"Save Changes"** button
7. System confirms: "Product updated successfully"
8. Changes are now live on the platform

### Sub-Module 2.4: Update Product Lifespan (Extend Expiry)

**Step-by-Step: Extending Product Lifespan**
1. From Products list, click on a product with expiring stock
2. Click **"Update Lifespan"** button
3. A form appears asking for:
   - New lifespan in days
4. Enter the number of days to extend
5. Click **"Confirm"** button
6. System updates the expiry date
7. Success message: "Product lifespan extended successfully"

### Sub-Module 2.5: Approve/Reject Products

**Step-by-Step: Approving New Products from Sellers**
1. From Products page, find products with **"Pending"** status
2. Click on the pending product
3. Review the product details carefully
4. Click **"Approve"** button
5. A confirmation dialog appears
6. Click **"Yes, Approve"** to confirm
7. Product status changes to **"Approved"**
8. Product is now visible to customers on the platform
9. Seller receives notification about approval

**Step-by-Step: Rejecting Products**
1. From Products page, find the product to reject
2. Click on the product details
3. Click **"Reject"** button
4. A form appears asking for rejection reason
5. Select or type the reason:
   - "Inappropriate content"
   - "Out of policy"
   - "Quality issues"
   - "Other reason" (specify)
6. Click **"Submit"** button
7. Seller receives notification with rejection reason
8. Product is removed from public view

### Sub-Module 2.6: Delete/Remove Product

**Step-by-Step: Removing Product from Platform**
1. From Products list, click on the product to remove
2. Click **"Delete Product"** button
3. A confirmation dialog appears: "Are you sure you want to delete this product?"
4. Additional option: "Notify seller about removal" (recommended)
5. Click **"Yes, Delete"** to confirm
6. Product is removed from the platform
7. Seller receives notification (if option was selected)
8. Success message: "Product deleted successfully"

## Module 3: Order Management

### Sub-Module 3.1: View All Orders

**Step-by-Step: Accessing Orders Dashboard**
1. From Admin Dashboard, click **"Orders"** in the sidebar
2. You will see a list of all orders placed on the platform
3. For each order, you can see:
   - Order ID
   - Customer name
   - Order date
   - Total amount
   - Number of items
   - Current delivery status
   - Assigned rider (if any)
   - Payment status

### Sub-Module 3.2: Filter Orders by Date Period

**Step-by-Step: Viewing Orders from Specific Time Period**
1. From Orders page, look for the **"Filter by Date"** section
2. Select time period from dropdown:
   - **Today**: Orders placed today
   - **This Week**: Orders from current week (Monday-Sunday)
   - **This Month**: Orders from current month
   - **This Year**: Orders from current year
   - **Custom Date Range**: Select specific start and end dates
3. Click **"Apply Filter"** button
4. Orders list will update to show only orders from selected period
5. You can see the filter status displayed at the top
6. Click **"Reset Filter"** to see all orders again

### Sub-Module 3.3: View Order Details

**Step-by-Step: Viewing Complete Order Information**
1. From Orders list, click on an order ID or **"View Details"** button
2. Order details page opens showing:
   - **Order Information**:
     - Order ID
     - Order date and time
     - Payment status (Pending / Paid / Failed)
     - Payment method (Online Payment / Cash on Delivery)
   - **Customer Information**:
     - Name
     - Email
     - Contact number
     - Delivery address
   - **Order Items**:
     - Product name
     - Product image
     - Seller name
     - Quantity ordered
     - Unit price
     - Item subtotal
   - **Pricing Summary**:
     - Subtotal
     - Shipping fee
     - Total amount
   - **Delivery Information**:
     - Assigned rider (if any)
     - Current delivery status
     - Expected delivery date
   - **Action Buttons**: Update status, assign rider, etc.
3. Review all information
4. Click **"Back"** to return to orders list

### Sub-Module 3.4: Assign Rider to Order

**Step-by-Step: Assigning Delivery Rider to Order**
1. From Order Details page, click **"Assign Rider"** button
2. A modal/dialog appears showing list of available riders
3. You will see for each rider:
   - Rider name
   - Contact number
   - Current status (Active/Inactive)
   - Number of pending deliveries
4. Select a rider from the list by clicking on them
5. Selected rider will be highlighted
6. Click **"Confirm Assignment"** button
7. System confirms: "Rider assigned successfully"
8. Order status updates to show assigned rider
9. Both customer and rider receive notifications

### Sub-Module 3.5: Update Order Delivery Status

**Step-by-Step: Updating Order Status Through Delivery Process**
1. From Order Details, click **"Update Status"** button
2. A dropdown menu appears with available status options:
   - **Pending**: Order created, awaiting payment/pickup
   - **In Transit**: Order with rider, on the way to customer
   - **Out for Delivery**: Rider is nearby, about to deliver
   - **Delivered**: Order received by customer
   - **Cancelled**: Order was cancelled
3. Select the appropriate status
4. If updating to "Delivered", you may need to add:
   - Delivery timestamp
   - Delivery notes
5. Click **"Confirm Update"** button
6. System updates the order status
7. Customer and rider receive notifications of status change
8. Order updates in the system

### Sub-Module 3.6: Cancel Orders

**Step-by-Step: Cancelling Orders with Documentation**
1. From Order Details, click **"Cancel Order"** button (if order is eligible)
2. A form appears asking for:
   - Cancellation reason (dropdown):
     - "Out of Stock"
     - "Customer Request"
     - "Payment Issue"
     - "Delivery Issue"
     - "Other"
   - Additional notes/details
   - Upload supporting documentation (optional image/file)
3. Fill in the required information
4. Upload proof if needed (e.g., screenshot of refund)
5. Click **"Confirm Cancellation"** button
6. Order status changes to **"Cancelled"**
7. System initiates refund process (if applicable)
8. Customer receives cancellation notification and refund details
9. Success message: "Order cancelled successfully"

### Sub-Module 3.7: Manage Refunds

**Step-by-Step: Updating Refund Status**
1. From Order Details, scroll to **"Refund Information"** section
2. You can see:
   - Refund request status (Pending / Approved / Rejected / Completed)
   - Customer's refund method (Bank Transfer / GCash / Maya / etc.)
   - Refund amount
   - Refund proof/receipt
3. If refund is pending, click **"Update Refund Status"** button
4. Select new status:
   - **Approve**: Process the refund
   - **Reject**: Deny the refund (provide reason)
   - **Complete**: Mark refund as sent to customer
5. If rejecting, provide reason in text field
6. Upload proof of refund if completing (refund receipt)
7. Click **"Confirm"** button
8. Customer receives notification about refund decision
9. Status updates in the system

### Sub-Module 3.8: Archive & Unarchive Orders

**Step-by-Step: Archiving Completed Orders**
1. From Order Details of a completed order, click **"Archive Order"** button
2. A confirmation dialog appears: "Archive this order?"
3. Click **"Yes, Archive"** to confirm
4. Order moves to archived section
5. Order is no longer displayed in active orders list
6. Success message: "Order archived successfully"
7. Archived orders are kept for historical records

**Step-by-Step: Viewing & Unarchiving Orders**
1. From Orders page, click **"View Archived Orders"** link/button
2. You will see list of all archived orders
3. To unarchive an order:
   - Click on the archived order
   - Click **"Unarchive Order"** button
   - Confirm in the dialog
4. Order moves back to active orders list
5. Order is now visible in regular orders view

### Sub-Module 3.9: Track Order

**Step-by-Step: Tracking Order Location & Status**
1. From Order Details, click **"Track Order"** button
2. Order tracking page opens showing:
   - **Current Status**: Text and visual indicator
   - **Timeline**: All status changes with timestamps:
     - When order was placed
     - When payment was received
     - When rider picked up
     - Current location/status
     - Estimated delivery time
   - **Rider Information**:
     - Rider name
     - Rider contact number
     - Rider current location (if available)
   - **Live Map** (if integrated):
     - Shows rider's current location
     - Route to delivery address
3. Refresh page to see latest tracking information
4. Click **"Back"** to return to order details

## Module 4: Financial Management

### Sub-Module 4.1: View All Transactions

**Step-by-Step: Accessing Financial Transactions**
1. From Admin Dashboard, click **"Transactions"** in sidebar
2. Or navigate to **"Financial"** → **"All Transactions"**
3. You will see transaction dashboard with tabs:
   - **All Transactions**
   - **Seller Payments** (Online Farmers)
   - **Offline Farmer Payments**
   - **Rider Payouts**
4. Each tab shows relevant transactions
5. For each transaction, you can see:
   - Transaction ID
   - User/Rider name
   - Amount
   - Transaction date
   - Status (Pending / Completed / Failed)
   - Payment method
   - Transaction type (Payment / Payout)

### Sub-Module 4.2: View Seller Payment Transactions

**Step-by-Step: Reviewing Seller Earnings**
1. From Transactions page, click **"Seller Payments"** tab
2. You will see all payments made to online farmers/sellers
3. For each payment, you can see:
   - Seller name
   - Amount earned
   - Date of payment
   - Payment method (Bank Transfer / GCash / Maya)
   - Status (Completed / Pending)
   - Number of orders included
4. Click on a payment to see:
   - Orders included in this payment
   - Breakdown of earnings by product
   - Commission deductions (if any)
5. Use filters to sort by:
   - Date range
   - Status
   - Seller name
   - Amount range

### Sub-Module 4.3: View Rider Payout Records

**Step-by-Step: Checking Rider Earnings & Payouts**
1. From Transactions page, click **"Rider Payouts"** tab
2. You will see all payouts to riders
3. For each payout, you can see:
   - Rider name
   - Payout amount (total earnings)
   - Date range for earnings
   - Payout date
   - Payment method
   - Status (Paid / Pending / Failed)
   - Number of deliveries completed
4. Click on a payout to view:
   - Individual delivery earnings
   - Delivery dates
   - Commission breakdown
   - Deductions (if any)

### Sub-Module 4.4: Update Payout Status

**Step-by-Step: Processing Seller Payouts**
1. From Seller Payments section, find a pending payout
2. Click on the payout record
3. Click **"Update Payout Status"** button
4. Select new status:
   - **Pending**: Waiting to process
   - **Processing**: Currently being transferred
   - **Completed**: Funds sent to seller
   - **Failed**: Transfer failed
5. If status is "Completed", you can:
   - Upload proof of transfer (receipt/screenshot)
   - Add transfer reference number
6. Click **"Save"** button
7. Seller receives notification of payout status
8. Record updates in system

**Step-by-Step: Processing Rider Payouts**
1. From Rider Payouts section, find a pending payout
2. Click on the payout to view details
3. Click **"Process Payout"** button
4. Review:
   - Total amount to be paid
   - Rider's payment method (GCash/Bank/Maya)
   - Rider's wallet number
5. Click **"Confirm Payout"** button
6. System processes the transfer
7. Status updates to "Completed" (after successful transfer)
8. Rider receives notification and payment confirmation
9. You can upload transfer proof if needed

### Sub-Module 4.5: Manage QR Codes for Payment

**Step-by-Step: Adding Payment QR Codes**
1. From Financial section, click **"QR Codes"** or **"Payment Settings"**
2. Click **"Add QR Code"** button
3. A form appears asking for:
   - Payment method (GCash / Maya / Bank / Other)
   - QR Code name (e.g., "GCash - Juan Dela Cruz")
   - Upload QR Code image file
   - Account holder name
   - Account number (optional)
4. Upload the QR code image
5. Click **"Save QR Code"** button
6. System stores the QR code
7. Success message: "QR Code added successfully"
8. QR code is now available for customers to use for payments

**Step-by-Step: Viewing All QR Codes**
1. From QR Codes page, you will see all payment QR codes
2. Organized by payment method:
   - GCash QR codes
   - Maya QR codes
   - Bank QR codes
3. For each QR code, you can see:
   - Payment method
   - QR Code image
   - Account holder name
   - Date added
   - Edit/Delete buttons

**Step-by-Step: Updating QR Code**
1. From QR Codes list, click **"Edit"** button on a QR code
2. Form appears with current QR code details
3. You can update:
   - Payment method
   - QR Code image (upload new image)
   - Account holder name
4. Click **"Save Changes"** button
5. System updates the QR code
6. Success message confirms the update

**Step-by-Step: Deleting QR Code**
1. From QR Codes list, click **"Delete"** button on the QR code
2. Confirmation dialog appears: "Are you sure you want to delete this QR code?"
3. Click **"Yes, Delete"** to confirm
4. QR code is removed from the system
5. Customer will no longer see this payment option
6. Success message: "QR Code deleted successfully"

### Sub-Module 4.6: Manage Shipping Fees

**Step-by-Step: Viewing Current Shipping Fees**
1. From Financial section, click **"Shipping Fees"** 
2. You will see current shipping fee structure:
   - Base shipping fee
   - Distance-based fees (if applicable)
   - Weight-based fees (if applicable)
3. Current active fee is highlighted
4. You can see:
   - Fee amount
   - Areas covered
   - Effective date
   - Last modified date

**Step-by-Step: Updating Shipping Fees**
1. From Shipping Fees page, click **"Edit Shipping Fee"** or **"Update Fee"** button
2. Form appears with current fee information
3. You can modify:
   - Base shipping fee amount
   - Weight tiers and fees (if applicable)
   - Distance zones and fees (if applicable)
   - Effective date
   - Special areas or exclusions
4. Click **"Save Changes"** button
5. System confirms the update: "Shipping fee updated successfully"
6. New shipping fee applies to all new orders
7. Notification can be sent to users about fee changes

## Module 5: Offline Farmer Management

### Sub-Module 5.1: Upload Products for Offline Farmers

**Step-by-Step: Adding Products for Farmers Without Accounts**
1. From Admin Dashboard, click **"Offline Farmers"** or **"Upload Products"**
2. Click **"Add Offline Farmer"** button
3. First, search for existing farmer or create new one:
   - **Search**: Enter farmer name or ID
   - **Create New**: Fill in farmer details if new:
     - Full name
     - Contact number
     - Address
     - Preferred payment method (Bank/GCash/Maya)
4. Once farmer is selected/created, click **"Add Product"**
5. Fill in the product form:
   - Product name
   - Category (Fruits / Vegetables / Root Crops / Grains / Legumes)
   - Price per unit
   - Stock quantity
   - Weight (kg)
   - Description
   - Upload product image
   - Lifespan (days)
6. Click **"Save Product"** button
7. Product is added to the farmer's inventory
8. Product appears on platform under farmer's profile
9. Success message: "Product added successfully"

### Sub-Module 5.2: Manage Offline Farmer Orders

**Step-by-Step: Processing Offline Farmer Payments**
1. From Offline Farmers section, find the farmer
2. Click on farmer to view their orders and earnings
3. You will see:
   - Orders from their products
   - Total earnings
   - Outstanding payment due
4. Click **"View Transactions"** to see:
   - All orders placed
   - Earnings breakdown
   - Commission deductions
5. To process payment:
   - Click **"Process Payment"** button
   - Confirm the amount to pay
   - Select payment method
   - Click **"Send Payment"** button
6. System sends payment to farmer
7. Payment status updates to "Completed"
8. Farmer receives notification and payment details

## Module 7: Content Management (Announcements)

### Sub-Module 7.1: Create Announcement

**Step-by-Step: Creating Platform-Wide Announcements**
1. From Admin Dashboard, click **"Announcements"** in sidebar
2. Click **"Create New Announcement"** button
3. A form appears with fields:
   - **Title**: Announcement title (e.g., "New Seasonal Products Available")
   - **Description**: Announcement details and message
   - **Image**: Upload announcement image/banner
   - **Category** (optional): Type of announcement
   - **Valid From**: Start date for announcement visibility
   - **Valid Until**: End date when announcement expires
   - **Priority**: High / Normal / Low (determines display position)
4. Fill in all information
5. Upload announcement image/banner
6. Click **"Preview"** to see how it will appear to users
7. Click **"Publish"** button to make it live
8. Announcement immediately appears to all users
9. Success message: "Announcement published successfully"

### Sub-Module 7.2: View All Announcements

**Step-by-Step: Managing Announcements**
1. From Announcements page, you will see list of all announcements
2. For each announcement, you can see:
   - Announcement title
   - Publication date
   - Expiry date
   - Current status (Active / Expired / Scheduled)
   - Number of views
   - Edit/Delete buttons
3. Announcements are sorted by validity/date
4. Active announcements are highlighted

### Sub-Module 7.3: Edit Announcement

**Step-by-Step: Modifying Existing Announcements**
1. From Announcements list, click **"Edit"** button on announcement
2. Edit form appears with current information
3. You can modify:
   - Title
   - Description/message
   - Image (upload new one if needed)
   - Valid dates (extend or shorten visibility)
   - Priority level
4. Make your changes
5. Click **"Preview"** to verify changes
6. Click **"Update"** button
7. Changes are saved and immediately reflected
8. Notification appears: "Announcement updated successfully"

### Shopping Tips
- **Compare Prices**: Check prices from different sellers for same product
- **Read Reviews**: Check customer reviews before buying
- **Check Stock**: Verify product is in stock before ordering
- **Fresh Products**: Look for recently uploaded products (fresher items)

- **Save Address**: Save your address for faster checkout next time
- **Confirm Contact**: Make sure your contact number is correct
- **Track Order**: Use track feature to know when order arrives
- **Be Ready**: Be available for delivery to receive your order
- **Inspect on Delivery**: Check items immediately when rider delivers

### Quality & Returns
- **Photo Evidence**: Always take clear photos if product is damaged
- **Quick Replacement**: Request replacement within 48 hours of delivery
- **Document Everything**: Keep photos/receipts for refund/replacement claims
- **Communicate**: Message seller immediately if there's an issue
- **Be Honest**: Write fair and honest reviews

### Account Security
- **Strong Password**: Use complex password with mix of characters
- **Don't Share**: Never share your password or OTP with anyone
- **Verify Email**: Always verify your email address
- **Update Contact**: Keep your contact number current
- **Logout**: Logout on shared devices after using your account

## Glossary of Terms

- **SKU**: Stock Keeping Unit - unique product identifier
- **COD**: Cash on Delivery - pay when order arrives
- **QR Code**: Quick Response code - used for payments
- **Rider**: Delivery person who brings your order
- **Seller**: Farmer/merchant selling products
- **Refund**: Money returned to you
- **Replacement**: New product sent to replace damaged one
- **OTP**: One Time Password - verification code sent to your phone
- **Checkout**: Process of finalizing your purchase
- **Invoice**: Receipt showing order details and payment
- **Payout**: Payment sent to seller for sales
